Maintenance questions

How can I edit the content of these pages?

You can use several ways to get access to the source files behind the pages.

Attention: You can only access these files by using Internet Explorer

  1. Access via Windows Explorer:
    • Open Windows Explorer (e.g. by Windows Key + E) and type "\\troom.capgemini.com@SSL\DavWWWRoot\sites\vcc\devon" without quotation marks.
    • Now you get the overview over all source files.

  2. Access via SharePoint:
    • Click here or at the right top corner of every page to switch to Mircosoft SharePoint, where an overview over all available source files is given.
    • Left click "Actions" from the menu bar and select "Open with Windows explorer".
    • Now you are able to change the source files in the opening explorer window.

  3. Access via SharePoint, for frequent usage (if you have not installed Microsoft SharePoint Designer):
    • Once open the file list by accessing it with one of the two approaches above.
    • Change default program settings for the source files. Select the editor you would like to work with.
    • From now on you can use this easy approach:
      • Click here or at the right top corner of every page to switch to Mircosoft SharePoint, where an overview over all available source files is given.
      • Hover the file you want to change and left click the orange dropdown arrow.
      • Select "Edit in Microsoft SharePoint Designer" from the opened context menu.
      • The selected file will now open in your standard editor.

General questions

Where do I find more details? Where does discussion take place?

We are using the following infrastructure for the OASP:

  • github
    • repositories per variant (OSAP4J, OASP4JS, OASP4Net) with code and wikis (1 repo for the standard incl. docs, 1 for sample application, 1 for capgemini IP)
    • Wiki
    • repository for the OASP itself containing variant independent stuff (e.g. general methodology for specification, requirements, agile, etc.)
  • APPS Evolve! SharePoint (this one and VCC Java, VCC .NET, etc.)
    • TODO: Folder and directory structure for document templates and ready to use documents as well as un-approved input
  • Yammer, Devon Group
  • Capgemini Mailing List
  • CI
    • Jenkins
    • Sonarqube

How can I have a look? How can I join?

Until the repositories are public, you need to have a github account to even read the contents. This is an undesired situation but it is only temporary. You should do the following steps:

  • Create a free github account - use your real name or Capgemini Corp login for your github account so we can associate you with the account
  • Send an email to one of the following person as request to join. Please also state if you want to actively contribute to Java or JavaScript variant. We will later switch to pull-requests but so far we just give you direct write permissions
    • Jörg Hohwiller
    • Marek Matczak
    • Simon Spielmann

The OASP admin receiving your mail will then

  • Add you to the Capgemini mailing list
  • Add you to the readaccess team on github
  • Add you to the JavaDev or JsDev team as requested

Why are we using github and not XXX?

  1. We believe that a standard has to be open-source in order to gain attention and have success. Our customers can use it and receive updates and will not be trapped in a vendor lock-in when we publish it as open source. Github is the most popular open source platform.
  2. Github is the number one platform for social coding. Via seamless git usage, pull-requests, AsciiDoc support for Wiki and other features it is unrivaled compared to other solutions.

OASP related questions

Is the OASP licensed as Open Source?

The OASP is licensed as open source under the Apache Software License 2.0. There will also be add-ons and tools with more code and invest that will not be open sourced and licensed proprietary (agreement with our customers for free non-exclusive usage in the context of the project). This way we will keep our intellectual property (IP) for high-level invests.

Therefore we are using the brand "devon" for IP content and OASP in combination. The name "devon" will be used to promote this to our customers.

Why is the name OASP and not XXX? What is Devon?

For the name we had to choose something that fulfills the following requirements:

  1. The name has to be available as internet domain name (DNS) with TLD .org (or maybe .net).
  2. The name must not be registered as trademark or the like.
  3. The name should stand for what is behind the project (the terms open and standard should be addressed).

Various obvious names (e.g. QUASAR) did not match all of these criteria. In the end we had to make a decision and the content is what we wanted to focus on rather than waiting month or years to find a perfect name.

As a marketing name we are using "devon" that is a brand owned by Capgemini. The OASP is "only" the public part of devon and we can use the name OASP for university collaborations where we only focus on the open standard.
