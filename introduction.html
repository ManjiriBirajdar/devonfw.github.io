iCSD2015

iCSD2015 is the Industrialization Stream of work focused on providing tools and methods to improve our Custom Software Development engagements.

CSD2015 covers two main assets:

  • The Production Line
  • Devon/OASP

Both address those topics that are relevant for engineers and architects when building especially CSD software. Within iCSD2015 we will focus on Java, at least in the first step. Hereby they are two sides of the same coin and complete each other:

One example may highlight this relation:

Analysing software with a tool means the tool needs to "know" the tools, rules and the methodology along with the software is built to ensure for example that the tool correctly identifies the areas where a code generator has produced software. Otherwise the tool might report the unnecessary doubling of code. So the configuration as part of the Production Line must reflect the architec-ture guidance etc.

The same holds for the shared services provided both for the Production Line and for the Platform.
